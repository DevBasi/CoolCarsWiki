# Contributing to CoolCars

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **CoolCars**. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
*   **Check the existing issues** to see if the bug has already been reported.
*   **Use the Bug Report template** when opening a new issue.
*   **Be as detailed as possible** including server version, plugin version, and steps to reproduce.

### Suggesting Enhancements
*   **Check if the feature already exists** or has been requested.
*   **Use the Feature Request template**.
*   **Explain the use case** and how it benefits the community.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the test suite passes.
5.  Make sure your code lints.
6.  Issue that pull request!

## Styleguide
*   Use clear and descriptive commit messages.
*   Follow the existing code style in the project.
*   Keep your changes focused.

Thanks!
